
# 四个最关键节点的专属话术
> 红线全藏在这四个节点。每个给你「启动 / 拉回 / 验收」三句。

## 🔴 节点 1 · Play 远端加载+隔离 ＋ S3 存储边界（D1 AM，最难、先做）
**踩这里直接挂**：红线②（Play 写死本地）、红线③（存储用 fs）。

启动
```
读 CLAUDE.md + docs/06 + docs/05 + docs/01 + docs/10。做 Checkpoint 1：
compose 起 pg/redis/minio + minio-init 建桶设 CORS；写 src/lib/storage.ts（S3，唯一碰 fs 的文件）；
手放一个静态游戏 bundle 到 MinIO；做 Play 通用 loader：iframe.src 指向 MinIO 远端 URL 加载它。
Play 外壳的 Source 徽章/状态胶囊按 docs/10；顺手接好 design token 基础（globals.css 变量 + Tailwind theme + Plus Jakarta Sans）供后续 UI 复用。先 plan。
```
拉回（本节点最常见跑偏）
```
停。iframe 必须 src=远端 MinIO URL，不许 srcdoc/内联；sandbox 只给 allow-scripts，不许 allow-same-origin；
产物不许进 public/；所有 S3/fs 只能在 storage.ts。
```
验收（你亲自点）
```
打开 Play 给我看：Network 里 index.html/game.js 来自 localhost:9000（异端口）；
Elements 里 iframe sandbox 没有 allow-same-origin；页面有 Source 徽章；
MinIO 换掉对象刷新游戏会变；改坏 URL 出错误卡不白屏；grep 确认 fs 只在 storage.ts。
```

## 🔴 节点 3 · 异步 ＋ 独立 worker ＋ 多 Agent 可见（D2 AM，AI Native 核心）
**踩这里挂**：红线④（同步单次黑盒）、红线①（只有 CRUD 无生成链路）、红线④衍生（固定假数据）。

启动
```
读 docs/04 + docs/03 + docs/10。做 Checkpoint 3：BullMQ + 独立 worker 进程；POST /api/tasks 写库+enqueue 后立即 202；
worker 按 docs/04 跑节点链，每节点写 AgentLog + 发 SSE；ModelClient 默认 mock（输入敏感）；

```
