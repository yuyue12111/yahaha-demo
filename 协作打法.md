## 2. 会话节奏（怎么开一局、怎么收一局）

**一个 session 尽量只推进一个 checkpoint**（见 §5 两天剧本），上下文干净、易回滚。

**开局对我说**（模板）：
> 读 `CLAUDE.md` 和相关 `docs/`。现在做 **Checkpoint N：<名字>**。先列实现步骤（plan mode + ultrathink），我确认后再写。完成后自查 + 跑起来验证 + 提交。

**收局我会做**：跑通验证 → 自查对照雷区清单 → 自动提交（语义化 message）→ 用一句话告诉你「这步完成了什么、怎么验、有无遗留」。

**你收局要做**：按该 checkpoint 的「验收清单」（§4）亲自点一遍（尤其 Play 的 Network 证据），有问题立刻反馈。

## 3. 怎么给我下指令（prompt 打法）

**好指令 = 目标 + 约束 + 验收**。例：
> ✅「实现 Checkpoint 3 的 worker 状态机：按 `docs/04` 的 6 节点，每节点写 AgentLog 并发 SSE。先 plan，跑通 mock 模式产出一个 bundle 到 MinIO 我再看。」

**少用模糊指令**：
> ❌「把 Create 做好」——没有边界，我会猜，容易跑偏或过度实现。

**几个触发词（按需用）**：
- `plan mode`：让我先出计划、你确认再动手（大改/不确定时用）。
- `ultrathink`：让我多想一层（架构权衡、难点设计时用）。
- `用 Explore/agent 去查`：在大范围找代码/确认用法时让我并行搜。
- `先验证再说`：要求我跑起来给证据，而不是只说"应该可以"。

**评审/复查**：复杂改动后可让我 `/code-review` 自查 diff；或你直接说「对照 `docs/0X` 检查这步有没有踩雷」。

**纠偏**：发现我跑偏，直接说「停，回到 `docs/0X` 的契约，X 应该是 Y」。要让纠偏**长期生效**（跨 session），告诉我「记住：以后……」，我会写进项目记忆/`CLAUDE.md`。

## 4. 防雷区验收清单（每个 checkpoint 收尾必过）

对照 `CLAUDE.md §2` 的五条 Fatal，落到可操作的检查：

**Play（最高优先，必须肉眼验）**
- [ ] DevTools Network：`index.html`/`game.js`/`assets` 来自 `localhost:9000`（MinIO，**异端口**），不是 app 源或 `/public`。
- [ ] Elements：`<iframe sandbox="allow-scripts">`，**没有** `allow-same-origin`；`src` 是远端 URL；**不是** `srcdoc`。
- [ ] 页面有 "Source: <远端 URL>" 徽章。
- [ ] 在 MinIO 控制台换掉该对象 → 刷新游戏变了（证明真远端，免重部署）。
- [ ] 制造失败（断网/改坏 URL）→ 出错误卡，**不白屏**。

**存储**
- [ ] `grep -rn "from 'fs'\|require('fs')\|writeFileSync" src` 只在 `src/lib/storage.ts` 命中。
- [ ] `public/` 下没有游戏产物；上传与产物都在 MinIO 控制台可见。

**生成（Create）**
- [ ] `POST /api/tasks` **立即**返回 taskId（Network 看，不是 hang 几十秒）。
- [ ] worker 是**独立进程**在跑（不是 API 请求里同步跑）。
- [ ] 步骤流可见：≥3 个节点各有独立 IO 摘要 + 状态流转。
- [ ] 换 prompt → 产物不同（mock 也要变）；任务有 failed 态 + retry。

**Auth / Home / 复现**
- [ ] 登录后 F5 仍在线；登出态访问 `/create` 跳登录。
- [ ] Home 来自 `GET /api/games?status=published`（Network 看），非写死数组。
- [ ] 干净环境能起；`.env.example` 无真密钥；无 key 也能 mock 跑通。

## 5. 两天剧本（每半天一个可跑 checkpoint）

> 与 `CLAUDE.md §7` 对齐。每个 checkpoint 末尾按 §4 验收 + 自动提交。落后了按 §6 砍。

| **D1 AM** | 1 · 存储 + Play 远端加载骨架 | 「先打通最难点：compose 起 pg/redis/minio + minio-init 建桶设 CORS；写 `storage.ts`；手放一个静态 bundle 到 MinIO；做 Play 跨域 sandbox 加载它，含 loading/loaded/failed。」 | Play 的 Network + iframe 证据（§4 Play 全项） | feat: storage + remote-load play |
| **D1 PM** | 2 · Auth + DB + Home | 「Prisma schema 按 `docs/02`；NextAuth 邮箱注册/登录/登出，F5 抗刷新，`/create` 受保护；Home 查库列表；seed 2 个预制游戏。」 | F5 在线、受保护跳转、Home 查库 | feat: auth + home browse |
| **D2 AM** | 3 · 异步多 Agent Create | 「BullMQ + 独立 worker；`POST /api/tasks` 202 立返；按 `docs/04` 跑 6 节点 + ModelClient(mock 默认)；per-step AgentLog + SSE；Create 多模态输入 + presigned 上传 + 预览。」 | 异步立返、步骤流可见、产物进 MinIO、异 prompt 异产物 | feat: async multi-agent generation |

## 6. 时间盒与降级（落后怎么办）

**判断**：每个半天结束若 checkpoint 没过，就是落后信号。**对我说「我们落后了，按降级阶梯砍 X」**，我执行并记到完成度说明。

**砍的顺序**（`CLAUDE.md §10`，从无痛到肉痛）：
Chromium 截图→SVG 缩略图 → 真模型默认→mock 默认（留变化逻辑+真实 seam） → plus 功能（搜索/标签/点赞/收藏/统计/详情/Remix） → SSE→轮询 → Agent 6 节点→3 节点 → OAuth 实现→只留设计+`Account` 表。

**绝不砍**（砍了直接挂）：真 MinIO+S3 SDK 无 fs · 独立 worker 202 立返 · ≥3 个有 IO 的 Agent 步骤 · 跨域 sandbox iframe 从 MinIO 加载+Source 徽章+非白屏 · ≥3 seed（≥1 真 Create）· README+`.env.example`(无密钥)+一条 compose · session 抗 F5。

## 7. 提交与版本

- **自动提交**：我在每个 checkpoint 完成且不破坏构建时**自动提交**（语义化 message + Co-Authored-By 尾注），你无需逐次点头。已写进 `CLAUDE.md §8`。

- **推送 GitHub** 仍需你确认（远端 `yuyue12111/yahaha-demo` + 鉴权）。要推时说「推到远端」，我先确认 remote 再 push。

- 出错回滚：`git log --oneline` 找点 → 跟我说「回到 <commit>」，我帮你 `git revert`/`reset`（破坏性操作前会跟你确认）。

## 8. 验证手段速查（本机无 Docker 下我怎么给你证据）

- 后端服务：brew 起 `postgresql@16` / `redis` / `minio`，`mc` 建桶设 CORS；`pnpm dev`(web) + `pnpm worker`(worker)。
- 接口：`curl` / 截 Network 给你看（202、查库、presign）。
- Play：浏览器工具打开 `localhost:3000` 截 Network + Elements，或给你点击脚本你自验。
- 生成：跑 mock 任务，给你看 MinIO 里新出现的 `games/<id>/<v>/` 对象 + AgentLog 步骤流。
- 你拿到成果后，最终演示请在**你有 Docker 的机器**上 `docker compose up` 跑一遍（评审实际跑的就是它）。
