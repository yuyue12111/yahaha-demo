## 3. 启动命令（必选）

```bash
# 1) 准备环境变量（无需任何真实密钥；默认 mock 即可跑通全链路）
cp .env.example .env

# 2) 一条命令起全部
docker compose up --build
```

一条 `docker compose up` 起全部依赖并自动完成初始化：

| 服务 | 作用 |
|---|---|
| `db` (Postgres 16) | 数据库 |
| `redis` | BullMQ 队列 |
| `minio` | 对象存储（S3 兼容），产物/沙箱源（`:9000`） |
| `minio-init` | 自动建桶 `yahaha` + 设 CORS + `games/` 公读 |
| `seed` | `prisma migrate deploy` 迁移 + 插 2 预制游戏 + **真跑一次 Create 流水线**产第 3 个并发布 |
| `worker` | 独立进程消费生成任务（6 节点 Agent 链） |
| `web` | Next.js 前后端（`:3000`） |

- **无需任何密钥**：默认 `MODEL_PROVIDER=mock`，离线可复现整条链路（确定性、异输入异产物）。
- **切真实模型**：在 `.env` 填 `MODEL_PROVIDER=openai` + `MODEL_BASE_URL` + `MODEL_API_KEY`（OpenAI 兼容端点 / Yahaha GPT-5.5）即可，**零改代码**。

## 8. 完成度说明（必选）

### ✅ 已完成（真实，非 Mock）
- **登录注册**：邮箱注册/登录/登出、JWT session 抗刷新、受保护页（`/create`、`/me`）双守卫；**OAuth**（Google/GitHub env-gated + **本地 Demo OAuth IdP** 无密钥端到端演示授权回调+账号绑定）。
- **Home**：查库展示所有已发布游戏（封面/标题/作者/简介/标签/发布时间）+ 详情页 + 搜索 + 标签筛选 + **点赞/收藏** + 游玩次数统计 + 三排（玩家之选/Trending/推荐）。
- **Play**：据 DB meta 动态解析 active version → 从 **MinIO 远端**加载 bundle，**跨域 sandbox iframe**（`allow-scripts` 不带 `allow-same-origin`）运行 + **Source 徽章** + postMessage 生命周期 + 非白屏失败态 + 加载优化（preconnect）。
- **Create**：多模态输入（文字 + 图片上传，真像素经 vision 影响生成）→ `POST /api/tasks` **202 立返** → **独立 worker** 异步 6 节点生成 → per-step **AgentLog** + SSE run-timeline → 产物上传 MinIO + meta 入库 → 预览/发布。**生成任务历史**（`/me?tab=tasks`，含 token 成本统计 + 失败重试）、**Remix 派生**、**版本管理**、**自然语言微调**（refine：一句话改现有游戏出新版本）。
- **安全**：跨域沙箱隔离、产物 CSP（`connect-src 'none'`）、颜色白名单、上传限额/签名绑长度、per-user 速率限额（Redis）、**内容审核**（生成前禁词拦截）、站点安全 headers、失效会话存在性再校验。
- **可复现**：一条 `docker compose up`、迁移自动应用、seed 真跑 Create、`.env.example` 无密钥、**无 key 也能 mock 跑通**。

### 🟡 Mock / 可选开关（诚实口径）
- **默认 `CODER_MODE=template`**：CODER 用**确定性引擎**（秒级、验收快稳）；模型仍参与 PLANNER 出 spec（配色/标题/玩法随输入变）。`CODER_MODE=auto` 开启后真 GPT-5.5 **真写整个 game.js**（已实测生成「地铁跑酷」复杂游戏），但慢 ~1–2min，仅 demo 用。
- **模型默认 mock**：无密钥时全链路确定性 mock；这是红线⑤要求的可复现保证，**非固定假数据**（mock 按 `(prompt+assets)` 哈希做种 → 异输入异产物，且真模型 seam 已接通）。
- OAuth：Google/GitHub 需真实 OAuth App 凭据才能真跑；**Demo OAuth IdP** 已可无密钥端到端演示同一套回调+绑定代码路径。

## 10. AI 协作记录（可选，已附）

- **AI 工具**：Claude Code（Opus 4.x），全程主力开发 + 自我审计。
- **关键 prompt 策略**：① 以 `CLAUDE.md` 为主规则 + `docs/` 契约为单一真源，每轮先读规则再改代码；② 五条 Fatal 红线写进规则、每个 checkpoint 对照倒查；③ ultracode 多 Agent 并行审计（18–41 agent + 对抗验证），把"高危发现"诚实降级（多数不碰红线）。
- **AI 贡献比例**：架构设计、代码实现、文档、测试与验收脚本 **~95%** 由 AI 完成；人工主要做需求拍板、方向纠偏（如"游戏长一样""微调""验收要快"）与 PR/合并。
- **review / test 方法**：每个 checkpoint 跑 `tsc` + `docker compose` 冷启动 + Chrome 驱动 live 验收（截图/DOM 取证）+ worker 内探针直跑产线函数（如 `runIngest`/`runGeneration`）+ 多 Agent 对抗审计。
- **人工修过的典型问题**：① "所有游戏长一样" → 定位为 CODER 是模板（非模型），引入 code-gen 后又因太慢改回模板默认；② 草稿"查看产物"GAME_NOT_FOUND → 修链接 + 加发布入口；③ 入场动画在搜索时重播 → 路径门控 + 存储语义修正；④ 真模型 code-gen 超时 → 单次调用 AbortController + 预算回退。
