# 00 · 产品总览与范围

## 产品目标

面向普通玩家与互动内容创作者的 AI Native 互动游戏平台：玩家发现并游玩社区发布的互动游戏；
创作者通过自然语言创意 + 多模态素材，与 AI Agent 协作生成可发布、可游玩的互动游戏。

## 角色与旅程

- **玩家（发现 → 游玩）**：未登录访客 → 浏览首页游戏流 → 点击 → Play 页 → 动态加载远端文件 → 开始游玩。
- **创作者（创意 → 发布）**：注册/登录 → Create 输入创意+上传素材 → 生成任务 → 看进度与结果 → 预览 → 发布 → 首页可见 → 被其他玩家点击 → 汇入同一 Play 页。
- **平台维护者**：关注生成任务稳定性、管理元信息与 OSS 文件、处理不安全内容/异常包/失败任务。

## 关键端到端链路（至少 1 个游戏必须真跑通且可复现）

```
注册 → 登录 → 刷新仍在线 → 受保护 /create
  → 输入创意 + 上传素材（进对象存储、被任务引用）
  → POST /api/tasks 立即返回 taskId（异步、非阻塞）
  → 独立 worker 跑多 Agent（INGEST→PLANNER→ASSET_CURATOR→CODER→VALIDATOR→PACKAGER），每步有状态流转 + 可读日志
  → 真实产物（index.html + manifest.json + assets）上传 MinIO 版本化路径 + meta 写库
  → 预览（从远端 URL 加载）→ 发布
  → Home（查库，非写死数组）出现卡片
  → 点击 → Play：loading → iframe.src=远端 MinIO URL → 跨域 sandbox 隔离运行 → loaded 可交互
  → 失败有错误态（不白屏）；结束有重开/返回
```

## 范围：MVP（必做）vs 加分（plus）

| 模块 | MVP 必做 | 加分（按降级阶梯可砍） |
|------|----------|------------------------|
| Auth | 邮箱注册/登录/登出、session 抗刷新、`/create` 受保护、OAuth 数据模型+设计 | 真实跑通 Google/GitHub 之一 |
| Home | 查库展示已发布游戏，卡片含封面/标题/作者/简介/标签/发布时间，点击进 Play | 详情页、搜索、标签筛选、点赞、收藏、游玩次数 |
| Play | 动态加载远端 manifest/bundle 并隔离运行；loading/loaded/failed/ended 四态 | 埋点统计、加载速度优化 |
| Create | 多模态输入（文字+文件/图/视频）、异步多 Agent 生成、上传对象存储、meta 写库、任务态、Agent 过程可见、预览发布 | 任务历史、失败重试、版本管理、Remix、内容审核、限额、成本统计 |

## 评分映射（推测权重，资源分配依据）

| 区域 | 权重 | 集中的自动否决项 |
|------|:----:|------------------|
| 闭环可跑 + 可复现 | ~22% | 3.4.5 |
| Create 多 Agent 异步链路 | ~22% | 3.4.1 / 3.4.4 / 异步+多Agent 约束 |
| Play 远端加载 + 隔离 | ~20% | 3.4.2 / 3.4.3(Play 侧) |
| S3 存储边界 | ~12% | 3.4.3 |
| Auth | ~8% | — |
| Home/浏览 | ~6% | — |
| 系统设计文档 | ~7% | — |
| 复现卫生/交付项 | ~3% | 3.4.5 |

→ 三大块（闭环/Create/Play）占 ~64% 且含 4/5 个自动否决项。资源优先级与构建顺序据此排定（见 `CLAUDE.md` §7）。

## 不接受的交付（3.4，必须全部规避，守卫见 `CLAUDE.md` §2）

1. 只有 CRUD，无 Create Agent 生成链路。
2. Play 只跑本地写死组件，无动态远端加载。
3. 对象存储用本地文件管理替代。
4. 生成全是固定假数据，无扩展到真实模型/Harness 的叙事。
5. 没 README / 起不来 / 复现不了。

## 交付物清单（3.3）

源码仓库（≥3 commits）· Demo 启动方式（本地 compose）· 启动命令 · 测试数据（≥3 游戏，≥1 来自 Create）·
`.env.example`（无密钥）· 系统设计文档（本 `docs/`）· 技术栈表 · 完成度说明 ·〔可选〕测试证据 / 演示视频 / AI 协作记录。

## 已知问题与限制（持续更新；对应 3.3「已知问题」交付项）

> 实现期间在此累积，交付前同步进 README 完成度说明。当前为初始已知项：

- Prompt Injection 仅做基本边界，未做深度防御/内容审核（设计见 `07-security.md`，列为后续）。
- 沙箱为浏览器 iframe 隔离，非服务端 VM/容器级隔离；headless 渲染（如启用）需受限无网环境。
- 确定性 mock 产出的游戏复杂度低于真实模型；演示真实质量需配置 `MODEL_*`（GPT-5.5）。
- 成本统计 / 速率限额 / Remix / 内容审核为加分项，按降级阶梯可能仅给设计不实现。
- 交付目标为本地 `docker compose`，无公网 URL（已与需求方确认，符合「本地完整启动方式」交付选项）。
